## Commit Conventions

- Use conventional commit messages for all future commits.
- Prefix each commit subject with one of: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `ci`, or `build`.
- Use the format:

```text
type(scope): short imperative summary
```

Examples:
- `feat(ci): add windows release workflow with changelog`
- `fix(bench): include local-error ping results in benchmark run eligibility`
